# 일용직 문서 생성 운영 체크리스트

- 작성일: 2026-03-23
- 대상: 현장 담당자, 사무 담당자, 관리 운영자
- 버전: v1.0

---

## A. 계약 생성 전 준비 체크리스트

계약 생성 버튼을 누르기 전에 반드시 아래 항목을 확인한다.

### A-1. 회사 정보 확인

관리자 메뉴 → 업체 관리에서 아래 값이 입력되어 있는지 확인한다.

```
□ 회사명
□ 대표자명
□ 사업자등록번호
□ 사업장 주소
□ 연락처 (전화번호)
```

> 회사 정보가 미등록 상태면 계약서에 빈칸이 출력됩니다.
> 계약 생성 전에 업체 마스터를 먼저 완성하세요.

---

### A-2. 근로자 정보 확인

관리자 메뉴 → 근로자 관리에서 아래 값이 입력되어 있는지 확인한다.

```
□ 성명
□ 연락처
□ 생년월일 (계약서 기재 필수)
□ 은행명
□ 계좌번호
□ 예금주
```

> 은행/계좌정보는 자동채움 되더라도 생성 전 최종 확인이 필요합니다.
> 계좌정보가 잘못되면 임금 지급 사고로 이어질 수 있습니다.

---

### A-3. 현장 정보 확인

관리자 메뉴 → 현장 관리에서 아래 값이 입력되어 있는지 확인한다.

```
□ 현장명
□ 현장 주소
```

---

### A-4. 계약 입력값 확인

```
□ 근로일 (workDate) — 반드시 입력. 누락 시 계약 효력 모호.
□ 일당 (dailyWage)
□ 공사명
□ 공종
□ 직종
□ 시업/종업 시각
□ 휴게 시작/종료 시각
```

> **근로일(workDate)은 일용직 계약에서 가장 중요한 필드입니다.**
> 누락하면 계약서에 "언제 일한 것인지" 명시되지 않습니다.

## B. 계약 생성 절차

1. 관리자 메뉴 → 계약 관리 → [신규 계약 등록]
2. **1단계: 계약 유형 분류**
   - 일용직의 경우 "직접고용" 선택
3. **2단계: 계약서 유형 선택**
   - "건설 일용근로자 근로계약서 (기본)" 선택
4. **3단계: 회사 선택**
   - 드롭다운에서 해당 회사 선택
   - 회사명·전화번호·사업자번호·주소·대표자가 자동 채움됨
   - 자동채움 값을 눈으로 확인할 것
5. **4단계: 근로자 선택**
   - 드롭다운에서 해당 근로자 선택
   - 생년월일·은행명·계좌번호·예금주가 자동 채움됨
   - 은행·계좌 정보를 반드시 눈으로 확인할 것
6. **5단계: 현장 선택**
   - 드롭다운에서 해당 현장 선택
   - 현장 주소가 자동 채움됨
7. **6단계: 계약 내용 입력**
   - 근로일 (workDate) 입력 — 필수
   - 공사명, 공종, 직종 입력
   - 일당 입력
   - 시업/종업/휴게 시각 확인 (기본값: 08:00~17:00, 휴게 12:00~13:00)
8. **[저장] 클릭**
9. 저장 성공 후 계약 상세 화면으로 이동

## C. 문서 생성 절차

계약 저장 후 아래 5종 문서를 생성한다.

### C-1. 근로계약서 생성

```
계약 상세 화면 → [근로계약서 생성] 버튼 클릭
→ 생성된 문서 텍스트 확인
→ 인쇄 또는 다운로드
```

### C-2. 부속문서 4종 생성

계약 상세 화면 또는 근로자 상세 → 안전문서 탭에서 아래 docType으로 각각 생성:

| 문서명 | docType | 필수 extraData |
|--------|---------|----------------|
| 근로조건 설명 확인서 | WORK_CONDITIONS_RECEIPT | 없음 (자동채움) |
| 개인정보 수집·이용 동의서 | PRIVACY_CONSENT | 없음 |
| 건설업 기초안전보건교육 확인서 | BASIC_SAFETY_EDU_CONFIRM | eduCompletedYn, eduCompletedDate, eduOrganization, confirmerName |
| 현장 안전수칙 준수 확인서 | SITE_SAFETY_RULES_CONFIRM | confirmerName |

> **기초안전보건교육 확인서**는 근로자의 이수 여부, 이수일, 교육기관명, 확인자 성명을 직접 입력해야 합니다.

## D. 생성 후 확인 항목

문서가 생성되면 반드시 아래를 확인한다.

### D-1. 자동채움 값 확인

```
□ 회사명 정확
□ 회사 연락처 정확
□ 근로자 성명 정확
□ 근로일 (workDate) 정확
□ 은행명 정확
□ 계좌번호 정확
□ 현장명 정확
□ 현장 주소 정확
□ 일당 금액 정확
```

### D-2. 문서별 특이 확인 항목

| 문서 | 확인 항목 |
|------|-----------|
| 근로계약서 | 4조 계약기간 — "1일 단위 근로" 문구 포함 여부 확인 |
| 근로조건설명확인서 | 일급 금액, 임금지급일, 시업/종업 시각 확인 |
| 개인정보동의서 | 동의함/동의하지 않음 체크박스 구조 확인 |
| 기초안전보건교육확인서 | 이수여부, 이수일, 교육기관명 정확 입력 여부 확인 |
| 현장안전수칙준수확인서 | 현장명, 확인자(현장소장) 성명 확인 |

### D-3. 서명란 확인

```
□ 회사(사용자) 서명란 존재
□ 근로자 서명란 존재
□ 서명란에 성명이 올바르게 출력됨
```

### D-4. 금지 문구 자가 점검

아래 문구가 없는지 확인한다.

```
□ "계속 고용"을 보장하는 표현 없음
□ "상시근로" 표현 없음
□ "무기계약" 또는 "정규직" 표현 없음
□ 월급제 또는 고정급 표현이 일용직 계약서에 포함되지 않음
```

## E. 오류 발생 시 확인 순서

### E-1. 저장 실패

```
1. 에러 메시지 확인 (화면 상단 빨간 박스)
2. 필수값 누락 확인:
   - 근로자 선택 여부
   - 시작일 입력 여부
   - 일당 입력 여부
   - 계약서 유형 선택 여부
3. 팀장형 차단 메시지인 경우:
   - 계약 유형을 "직접고용"으로 변경
```

### E-2. 문서 생성 실패

```
1. 계약 ID 확인 (URL에 표시)
2. docType 오타 확인
3. extraData 필수값 확인 (BASIC_SAFETY_EDU_CONFIRM은 eduCompletedYn 필요)
4. 서버 상태 확인 (관리자에게 문의)
```

### E-3. 자동채움 빈칸 발생

```
1. 근로자 마스터에 해당 정보가 입력되어 있는지 확인
2. 회사 마스터에 연락처·주소가 입력되어 있는지 확인
3. 현장 마스터에 주소가 입력되어 있는지 확인
4. 값 입력 후 계약을 다시 생성
```

## F. 문서 5종 중 반드시 출력해야 하는 상황

| 문서 | 반드시 출력하는 상황 |
|------|---------------------|
| 근로계약서 | 모든 일용직 근로자 고용 시 — **필수** |
| 근로조건 설명 확인서 | 근로자에게 근로조건을 설명하고 수령 확인이 필요할 때 — **필수** |
| 개인정보 수집·이용 동의서 | 근로자 개인정보 수집 전 — **필수** |
| 기초안전보건교육 확인서 | 건설현장 최초 투입 시 / 이수증 미확인 근로자 투입 전 — **필수** |
| 현장 안전수칙 준수 확인서 | 현장 배치 시 — **필수** |

> **5종 모두 필수**입니다. 근로자 서명 후 원본 1부를 회사가 보관하고 사본 1부를 근로자에게 교부합니다.

## G. 운영 주의사항 요약

1. **계약 생성 전 마스터 정보 완성 원칙**
   - 계약 생성 후 마스터(회사/근로자/현장) 정보를 바꿔도 기존 계약의 스냅샷 필드는 바뀌지 않음
   - 단, workerName·workerPhone·siteName은 마스터 변경 시 반영됨 → 계약 전 확정 필요

2. **workDate는 반드시 입력**
   - 일용직 계약서에서 "언제 일했는지"를 특정하는 핵심 필드
   - 누락 시 계약 효력 범위가 모호해짐

3. **은행/계좌정보 최종 확인**
   - 자동채움 되더라도 숫자 오류가 있을 수 있음
   - PDF 생성 후 본문에서 계좌번호를 직접 육안으로 확인

4. **PDF 생성 성공 ≠ 완료**
   - 201 응답을 받았어도 본문 내용과 서명란을 반드시 확인
   - 특히 금액·날짜·성명 오류는 서명 전에 수정해야 함
