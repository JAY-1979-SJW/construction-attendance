## 1. 전체 구조 요약

### 1.1 시스템 개요

```
현장 근로자 (스마트폰)
   └─ QR 스캔 → /qr/[qrToken]
         └─ 출근/퇴근 버튼
              └─ GPS 위치 검증
                   └─ AttendanceLog 생성/갱신

관리자 (PC/태블릿)
   └─ /admin/**
         ├─ 대시보드 (실시간 현황)
         ├─ 근로자 관리
         ├─ 현장 관리 + QR 발급
         ├─ 출퇴근 조회 + Excel 내보내기
         ├─ 예외 승인
         └─ 기기 변경 승인
```

### 1.2 기술 스택

| 계층 | 선택 | 이유 |
|------|------|------|
| Framework | Next.js 14 App Router | SSR/SSG/API 통합, 풀스택 단일 레포 |
| Language | TypeScript 5 | 타입 안전성, 자동완성 |
| DB | PostgreSQL 15 | 트랜잭션, JSON, 날짜 타입 지원 |
| ORM | Prisma 5 | 타입세이프 쿼리, 마이그레이션 |
| Auth | jose (JWT) + httpOnly Cookie | 모바일 브라우저 호환, XSS 방어 |
| SMS | CoolSMS / NCloud / Mock (env 전환) | 운영 유연성 |
| 검증 | Zod | 런타임 스키마 검증 |
| QR | qrcode (Node.js) | PNG 버퍼 생성, 서버사이드 렌더 |
| Excel | xlsx | 서버사이드 Excel 생성 |
| 프로세스 | pm2 | 무중단 재시작, 로그 관리 |
| 리버스 프록시 | Nginx | HTTPS, 캐싱, 보안 헤더 |

### 1.4 인증 플로우

```
[근로자 인증]
휴대폰 입력 → OTP 발송(SMS) → OTP 검증 → temp_token(Cookie)
   → 기기 등록 여부 확인
      ├─ 등록 있음: worker_token 발급 → /qr/* 접근 가능
      └─ 등록 없음: /device/register → deviceToken 저장 → worker_token 발급

[관리자 인증]
이메일+비밀번호 → bcrypt 검증 → admin_token(Cookie) → /admin/* 접근 가능

[미들웨어 보호]
/admin/* (단, /admin/login 제외) → admin_token 검증
/api/admin/* (단, /api/admin/auth/* 제외) → admin_token 검증
/api/attendance/*, /api/device/* → worker_token 검증
```

## 2. 정책 결정사항

### 2.1 날짜 기준 정책

| 항목 | 결정값 | 근거 |
|------|--------|------|
| 날짜 전환 기준 | **04:00 KST** | 야간 작업자(0~3시) → 전날로 처리 |
| 저장 단위 | `workDate DATE (KST)` | 시각은 checkInAt/checkOutAt 별도 |
| 타임존 처리 | 서버 UTC 저장, workDate는 KST 변환 후 계산 | DB 타임존 혼동 방지 |
| 자정 기준 | `(현재시각 - 4h).toDateString()` 로 workDate 결정 | 코드: `lib/utils/date.ts` → `getKstWorkDate()` |

### 2.2 출퇴근 기록 정책

| 항목 | 결정값 | 근거 |
|------|--------|------|
| 1일 1현장 1건 | **강제** (unique constraint) | `@@unique([workerId, siteId, workDate])` |
| 중복 출근 시 | **거부** (HTTP 409) | 혼선 방지, 예외는 관리자가 처리 |
| 퇴근 시간 없이 다음날 | 스케줄러가 `MISSING_CHECKOUT` 상태로 전환 | 관리자 검토 대상 |
| 사용자 자기 수정 | **불가** | 관리자만 수정/예외 처리 가능 |
| 퇴근 후 재출근 | **불가** (COMPLETED 상태면 거부) | 하루 1건 원칙 |

**AttendanceStatus 전이도:**
```
WORKING → (퇴근 성공) → COMPLETED
WORKING → (다음날 04:00 미퇴근) → MISSING_CHECKOUT
WORKING/MISSING_CHECKOUT → (관리자 조정) → ADJUSTED
WORKING/MISSING_CHECKOUT → (관리자 수동 입력) → ADMIN_MANUAL
```

> **주의:** 현재 schema의 AttendanceStatus에 `MISSING_CHECKOUT`, `ADJUSTED`, `ADMIN_MANUAL`이 빠져 있음.
> 아래 3절 DB 설계에서 확장된 enum을 적용할 것.

### 2.3 GPS 정책

| 항목 | 결정값 | 비고 |
|------|--------|------|
| 기본 허용 반경 | 현장별 설정 (기본 100m) | Site.allowedRadius |
| GPS 검증 실패 시 | 출근 **거부** (기본) / 환경변수로 경고만 가능 | `GPS_STRICT_MODE=true` |
| GPS 정확도 임계값 | 50m 이하일 때만 허용 | 클라이언트에서 accuracy 체크 후 전송 |
| 실내/GPS 불가 예외 | 예외 신청 후 관리자 승인 | `/api/attendance/exception-request` |

### 2.4 기기 인증 정책

| 항목 | 결정값 |
|------|--------|
| 1인 1기기 | 기본 정책 (isPrimary 기기 1개) |
| 기기 변경 | 관리자 승인 후 처리 (DeviceChangeRequest) |
| 기기 토큰 생성 | 클라이언트 fingerprint (UA + 화면 해상도 + 언어) → SHA256 |
| 토큰 유효기간 | 만료 없음 (isActive 플래그로 관리) |
| 신규 기기 감지 | worker_token에 deviceToken 포함 → 매 요청 검증 |

### 2.5 OTP 정책

| 항목 | 결정값 |
|------|--------|
| OTP 자릿수 | 6자리 숫자 |
| 유효기간 | 5분 |
| 재발송 제한 | 1분 내 재발송 차단 (verifiedAt 없는 최근 레코드 확인) |
| 동일번호 중복 | 기존 미검증 OTP 무효화 후 신규 발급 |
| SMS 공급자 | `SMS_PROVIDER=mock|coolsms|ncloud` env로 전환 |

### 2.6 관리자 정책

| 항목 | 결정값 |
|------|--------|
| 비밀번호 해싱 | bcrypt rounds=12 |
| 세션 유효기간 | 8시간 (admin_token JWT) |
| 권한 체계 | SUPER_ADMIN > ADMIN > VIEWER |
| 감사 로그 | 모든 쓰기 작업 AdminAuditLog 기록 |
| 초기 관리자 | seed.ts에서 환경변수 `ADMIN_INIT_EMAIL`, `ADMIN_INIT_PASSWORD` 로 생성 |

### 3.4 workDate 계산 로직

```typescript
// lib/utils/date.ts
export function getKstWorkDate(now: Date = new Date()): Date {
  // KST = UTC+9, 04:00 KST 기준으로 날짜 전환
  const kstMs = now.getTime() + 9 * 60 * 60 * 1000
  const kstDate = new Date(kstMs)

  // 04:00 이전이면 전날로 처리
  if (kstDate.getUTCHours() < 4) {
    kstDate.setUTCDate(kstDate.getUTCDate() - 1)
  }

  // 날짜만 반환 (시각 제거)
  return new Date(
    Date.UTC(kstDate.getUTCFullYear(), kstDate.getUTCMonth(), kstDate.getUTCDate())
  )
}

export function kstDateStringToDate(yyyymmdd: string): Date {
  const [y, m, d] = yyyymmdd.split('-').map(Number)
  return new Date(Date.UTC(y, m - 1, d))
}
```

## 4. API 명세 상세

> 공통 응답 형식:
> ```json
> { "success": true/false, "data": {...}, "message": "..." }
> ```

### 4.1 인증 API

#### POST /api/auth/send-otp
```
Request:  { phone: string }  // 010-xxxx-xxxx 또는 01012345678
Response: { success: true, message: "OTP 발송 완료" }
Errors:
  400 - 전화번호 형식 오류
  429 - 1분 내 재발송 차단
  404 - 등록되지 않은 근로자 (phone 없음)
Logic:
  1. phone 정규화 (하이픈 제거)
  2. Worker 조회 → 없으면 404
  3. 1분 내 미검증 OTP 존재 여부 확인 → 429
  4. 기존 미검증 OTP 무효화 (verifiedAt = null인 건 삭제)
  5. 6자리 OTP 생성, expiresAt = now+5min
  6. OtpVerification 저장
  7. SMS 발송
```

#### POST /api/auth/verify-otp
```
Request:  { phone: string, code: string, purpose: "LOGIN" | "DEVICE_CHANGE" }
Response: {
  success: true,
  data: {
    workerId: string,
    workerName: string,
    hasDevice: boolean,
    needsDeviceRegister: boolean
  }
}
Cookie:   Set worker_temp_token (httpOnly, 10분, purpose=LOGIN일 때)
Errors:
  400 - 코드 형식 오류
  401 - OTP 불일치 또는 만료
  404 - 근로자 없음
Logic:
  1. OtpVerification 조회 (phone + code + purpose + expiresAt > now)
  2. 없으면 401
  3. verifiedAt 업데이트
  4. Worker 조회
  5. 활성 기기(WorkerDevice.isActive=true) 존재 여부 확인
  6. temp_token 발급 (payload: { workerId, purpose, type: 'temp' })
  7. Set-Cookie: worker_temp_token
```

#### POST /api/auth/logout (근로자)
```
Response: { success: true }
Cookie:   worker_token 삭제 (maxAge=0)
```

#### GET /api/auth/me
```
Headers:  Cookie: worker_token
Response: { success: true, data: { workerId, name, company, jobTitle } }
Errors:   401 - 토큰 없음/만료
```

### 4.2 기기 관리 API

#### POST /api/device/register
```
Request:  { deviceToken: string, deviceName: string }
Headers:  Cookie: worker_temp_token (purpose=LOGIN)
Response: { success: true, data: { workerId, workerName } }
Cookie:   Set worker_token (httpOnly, 8h) + 삭제 worker_temp_token
Errors:
  401 - temp_token 없음/만료/purpose 불일치

#### M-01: 휴대폰 인증 (/login)

**목적:** 근로자 로그인 진입점

**컴포넌트:**
- 로고/타이틀 영역 (`해한 현장 출퇴근`)
- 전화번호 입력 필드 (숫자만, 자동 포매팅 `010-xxxx-xxxx`)
- `인증번호 받기` 버튼 (로딩 상태 포함)
- 에러 메시지 표시 영역

**상태 관리:**
```typescript
phone: string           // 입력값
isLoading: boolean
error: string | null
```

**동작:**
1. 전화번호 정규화 후 POST /api/auth/send-otp
2. 성공 → router.push(`/login/verify?phone=${encodeURIComponent(phone)}`)
3. 실패 → error 메시지 표시

**UX 고려:**
- 모바일 키보드: `inputMode="numeric"` 또는 `type="tel"`
- 이미 로그인 상태 → /qr/... 또는 /attendance 리다이렉트

---

#### M-02: OTP 입력 (/login/verify)

**목적:** 발송된 OTP 6자리 입력

**컴포넌트:**
- 안내 문구 (`${phone}으로 발송된 인증번호를 입력하세요`)
- OTP 입력 필드 6자리 (단일 input 또는 박스 6개)
- `인증하기` 버튼
- `재발송` 버튼 + 60초 카운트다운
- 에러 메시지

**동작:**
1. POST /api/auth/verify-otp
2. `hasDevice=false` → router.push('/device/register')
3. `hasDevice=true` → 원래 경로 또는 /attendance

**UX 고려:**
- 6자리 자동 제출 (입력 완료 시 즉시 API 호출)
- 재발송 버튼: 60초 쿨다운 타이머

---

#### M-03: 기기 등록 (/device/register)

**목적:** 최초 기기 등록 또는 신규 기기로 전환

**컴포넌트:**
- 안내 문구 (`이 기기를 출퇴근 전용 기기로 등록합니다`)
- 기기 이름 입력 (선택, 기본: `${navigator.userAgent 축약}`)
- 기기 토큰 자동 생성 표시 (숨김 또는 읽기전용)
- `이 기기로 등록하기` 버튼

**동작:**
1. `generateDeviceToken()` (lib/utils/device-token.ts) 실행
2. POST /api/device/register
3. 성공 → /attendance 또는 이전 경로

---

#### M-04: QR 출퇴근 (/qr/[qrToken])

**목적:** QR 스캔 후 출근/퇴근 처리 핵심 화면

**컴포넌트:**
- 현장명 + 주소 표시 (Site 정보)
- 현재 상태 카드:
  - `미출근`: 회색
  - `출근중 HH:MM 경과`: 파란색
  - `퇴근완료`: 초록색
- GPS 위치 수신 중 표시 (스피너)
- 현재 위치와 현장 거리 표시 (`현장까지 약 Xm`)
- `출근하기` / `퇴근하기` 버튼 (상태에 따라 전환)
- `GPS 문제` 버튼 → 예외 신청 모달
- 에러 표시

**상태 관리:**
```typescript
site: SiteInfo | null
todayLog: AttendanceLog | null
gpsState: 'loading' | 'ok' | 'error' | 'too_far'
position: { lat, lng, accuracy } | null
distance: number | null
isSubmitting: boolean
error: string | null
```

**동작 시퀀스:**
```
진입 → GET /api/sites/by-qr/[qrToken] (현장 확인)
     → GET /api/attendance/today (오늘 기록 확인)
     → navigator.geolocation.getCurrentPosition()
     → GPS 수신 완료
     → 출근 버튼 클릭 → POST /api/attendance/check-in
     → 퇴근 버튼 클릭 → POST /api/attendance/check-out
```

**예외 신청 모달:**
- 사유 입력 textarea
- `신청하기` → POST /api/attendance/exception-request

## 부록 A: 주요 유틸 함수 참조

### lib/gps/distance.ts — Haversine 공식
```typescript
export function haversineDistance(
  lat1: number, lng1: number,
  lat2: number, lng2: number
): number {
  const R = 6371000  // 지구 반지름 (m)
  const dLat = ((lat2 - lat1) * Math.PI) / 180
  const dLng = ((lng2 - lng1) * Math.PI) / 180
  const a =
    Math.sin(dLat / 2) ** 2 +
    Math.cos((lat1 * Math.PI) / 180) *
    Math.cos((lat2 * Math.PI) / 180) *
    Math.sin(dLng / 2) ** 2
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a))
}
```

### lib/utils/device-token.ts — 기기 토큰 생성
```typescript
export function generateDeviceToken(): string {
  const ua = navigator.userAgent
  const screen = `${window.screen.width}x${window.screen.height}`
  const lang = navigator.language
  const raw = `${ua}|${screen}|${lang}`
  // SHA256은 crypto.subtle 사용 (브라우저) 또는 Node crypto
  return btoa(raw).replace(/[^a-zA-Z0-9]/g, '').slice(0, 64)
}
```
