# 일용직 문서 생성 관리자 화면 사용 절차서

- 작성일: 2026-03-23
- 대상: 앱 관리자, 현장 사무 담당자
- 버전: v1.0

---

## 접속 방법

브라우저에서 `https://attendance.haehan-ai.kr/admin` 접속 후 관리자 계정으로 로그인.

---

## 1. 신규 일용직 계약 생성 절차

### 1-1. 메뉴 진입

```
좌측 메뉴 → [계약 관리] → [신규 계약 등록] 클릭
또는
URL: /admin/contracts/new
```

---

### 1-2. Step 1 — 계약 유형 분류 선택

화면 상단에 3가지 유형이 표시됨.

| 옵션 | 선택 기준 |
|------|-----------|
| **직접고용** | 회사가 출퇴근·공수·급여를 직접 관리하는 일용직 → **일용직은 이것 선택** |
| 외주팀 (사업자 있음) | 사업자등록이 있는 외주업체에 도급을 줄 때 |
| 팀장형 (사업자 없음) ⚠ | 사업자 없는 팀장·조장 구조 (법적 검토 필요 상태로 등록) |

> 일용직은 반드시 **직접고용** 선택.

---

### 1-3. Step 2 — 계약서 유형 선택

직접고용 선택 후 계약서 유형 드롭다운에서 선택:

```
건설 일용근로자 근로계약서 (기본)  ← 일용직은 이것 선택
상용직 근로계약서
기간제 근로계약서
[관리자] 월단위 기간제 근로계약서
[관리자] 계속근로형 근로계약서
```

> `[관리자]` 접두어가 붙은 유형은 내부 운영용. 일반 일용직에는 사용하지 않음.

---

### 1-4. Step 3 — 회사 선택

"회사 선택" 드롭다운에서 해당 회사를 선택.

**자동으로 채워지는 값:**
- 회사명
- 연락처
- 사업자등록번호
- 회사 주소
- 대표자명

> 자동채움 후 값을 눈으로 확인하고, 잘못된 경우 회사 마스터를 먼저 수정.

---

### 1-5. Step 4 — 근로자 선택

"근로자 선택" 드롭다운에서 해당 근로자를 선택.

**자동으로 채워지는 값:**
- 생년월일
- 은행명
- 계좌번호 (마스킹 표시)
- 예금주 (기본값: 근로자 성명)

> **은행/계좌번호는 임금 지급에 직결됩니다. 반드시 확인하세요.**
> 근로자 마스터에 은행정보가 없으면 빈칸으로 표시됩니다 → 근로자 관리에서 먼저 입력 필요.

---

### 1-6. Step 5 — 현장 선택

"현장 선택" 드롭다운에서 해당 현장을 선택.

**자동으로 채워지는 값:**
- 현장 주소

> 현장 마스터에 주소가 없으면 빈칸. 현장 관리에서 먼저 입력 필요.

---

### 1-7. Step 6 — 계약 내용 입력

아래 항목을 직접 입력한다.

| 항목 | 필수 | 기본값 | 설명 |
|------|------|--------|------|
| **근로일 (workDate)** | **필수** | 없음 | 오늘 날짜 또는 실제 근로일. 반드시 입력 |
| 공사명 | 권장 | 없음 | 현장 공사명 |
| 공종 | 권장 | 없음 | 전기/소방전기/기계설비 등 |
| 직종 | 권장 | 없음 | 보통인부/전공/기능공 등 |
| **일당** | **필수** | 없음 | 숫자만 입력 (예: 200000) |
| 시업 시각 | 자동 | 08:00 | 변경 필요 시 수정 |
| 종업 시각 | 자동 | 17:00 | 변경 필요 시 수정 |
| 휴게 시작 | 자동 | 12:00 | 변경 필요 시 수정 |
| 휴게 종료 | 자동 | 13:00 | 변경 필요 시 수정 |
| 임금지급일 | 권장 | 말일 | 25 또는 말일 등 숫자 입력 |
| 4대보험 | 선택 | 산재만 ON | 해당 항목 체크 |

---

### 1-8. 저장

입력 완료 후 **[저장]** 버튼 클릭.

- 성공 시: 해당 계약 상세 화면으로 자동 이동
- 실패 시: 화면 상단에 빨간 에러 메시지 표시 → 내용 확인 후 수정

---

## 2. 근로계약서 PDF 생성

계약 상세 화면에서:

```
[근로계약서 생성] 또는 [PDF 생성] 버튼 클릭
→ 텍스트 형식 문서 생성
→ 브라우저에서 복사/인쇄 가능
```

**생성 후 반드시 확인할 항목:**
- 제목: "건설 일용근로자 근로계약서"
- 1조 사용자 정보: 회사명·전화번호·주소 정확 여부
- 2조 근로자 정보: 성명·생년월일 정확 여부
- 3조 근로계약 내용: 근로일·현장명·공종·직종 정확 여부
- 4조 계약기간: "1일 단위 근로" 문구 포함 여부
- 7조 임금: 일급 금액 정확 여부
- 8조 임금 지급계좌: 은행명·계좌번호·예금주 정확 여부
- 서명란: 회사(사용자)·근로자 서명란 정상 여부

---

## 3. 부속문서 4종 생성

계약 상세 화면 또는 근로자 상세 → 안전문서 탭에서 각 문서를 생성한다.

### 3-1. 근로조건 설명 확인서

```
docType: WORK_CONDITIONS_RECEIPT
extraData: 없음 (전부 자동채움)
```

확인 항목:
- 현장명, 공종, 직종, 근로일 정확 여부
- 시업/종업, 휴게시간, 일급, 임금지급일 정확 여부
- 관리자 서명란과 근로자 서명란 존재 여부

---

### 3-2. 개인정보 수집·이용 동의서

```
docType: PRIVACY_CONSENT
extraData: 없음 (전부 자동채움)
```

확인 항목:
- 회사명 정확 여부
- 근로자 성명 정확 여부
- "[  ] 동의함 / [  ] 동의하지 않음" 체크박스 구조 존재 여부
- 근로자 서명란 존재 여부

---

### 3-3. 건설업 기초안전보건교육 확인서

```
docType: BASIC_SAFETY_EDU_CONFIRM
extraData:
  - eduCompletedYn: true/false (이수 여부)
  - eduCompletedDate: "YYYY-MM-DD" (이수일)
  - eduOrganization: "교육기관명"
  - confirmerName: "현장소장 성명"
```

> 이 문서는 extraData를 반드시 입력해야 합니다.
> 이수증 원본을 확인한 뒤 확인자(현장소장)가 서명합니다.

확인 항목:
- 이수여부 체크 표시 정확 여부
- 이수일, 교육기관명 정확 여부
- 확인자 성명 정확 여부
- 근로자 서명란 존재 여부

---

### 3-4. 현장 안전수칙 준수 확인서

```
docType: SITE_SAFETY_RULES_CONFIRM
extraData:
  - confirmerName: "현장소장 성명"
  - specialSafetyRules: "특이 안전수칙" (선택 입력)
```

확인 항목:
- 7개 안전수칙 항목 모두 표시 여부
- 현장명, 근로자 성명, 근로일 정확 여부
- 관리자 서명란과 근로자 서명란 존재 여부

---

## 4. 생성 완료 후 처리 절차

1. **출력**: 브라우저에서 인쇄 또는 PDF 저장
2. **서명 수령**: 현장에서 근로자에게 직접 서명 받음
3. **보관**: 원본 1부 회사 보관 / 사본 1부 근로자 교부
4. **보존 기간**: 근로관계 종료 후 3년 (근로기준법 제42조)

---

## 5. 자주 발생하는 문제 및 해결 방법

### 문제 1: 자동채움 값이 비어 있음

```
원인: 근로자/회사/현장 마스터에 해당 정보 미입력
해결:
  1. [근로자 관리] → 해당 근로자 → 편집 → 은행정보/생년월일 입력
  2. [업체 관리] → 해당 회사 → 편집 → 연락처/주소 입력
  3. [현장 관리] → 해당 현장 → 편집 → 주소 입력
  4. 계약 생성 화면으로 돌아와 재시도
```

### 문제 2: 저장 시 "일당 또는 월급 필수" 오류

```
원인: 일당 입력 필드가 비어 있음
해결: 일당 금액 입력 후 저장
```

### 문제 3: 팀장형 계약 차단 메시지

```
원인: 외주팀 계약 선택 + 직접관리 체크가 동시에 되어 있음
해결:
  - 직접 고용 근로자면 → "직접고용" 유형으로 변경
  - 실제 외주팀이면 → 직접관리 체크 해제
```

### 문제 4: 문서 생성 실패 (400 오류)

```
원인: 잘못된 docType 또는 extraData 누락
해결:
  - docType 철자 확인 (대문자, 언더스코어)
  - BASIC_SAFETY_EDU_CONFIRM은 extraData 필수 입력 확인
```

## 6. 계약 상태 흐름 참고

```
DRAFT (초안)
  → [계약 활성화] 클릭
ACTIVE (활성)
  → [계약 종료] 클릭
ENDED (종료)
```

> 일용직 계약은 1일 단위이므로 보통 DRAFT 상태로 문서 생성 후 당일 서명 처리합니다.
