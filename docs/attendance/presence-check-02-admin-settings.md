# 중간 체류 확인 기능 — 관리자 설정 명세 상세 지시문

> 기준일: 2026-03-21
> 대상 화면: `/admin/settings/attendance`

---

## 1. 화면 목적

- 현재 조직이 이 기능을 쓸 수 있는지
- 현재 기능이 켜져 있는지
- 알림 방식 (오전/오후 랜덤)
- 반경 및 응답시간
- 실패 처리 정책

---

## 2. 메뉴 위치

경로: `/admin/settings/attendance`
섹션명: **중간 체류 확인** (유료 옵션 기능)

---

## 3. 화면 상단 요약 카드

| 카드 | 표시 항목 |
|------|-----------|
| 현재 플랜 | 플랜명, 유료 여부, 기능 사용 가능 여부 |
| 현재 기능 상태 | 활성화 여부, 오전/오후 사용 여부 |
| 현재 정책 요약 | 반경, 응답 제한, 실패 처리 방식 |

---

## 4. 무료 플랜 표시

- 모든 토글 비활성화
- 모든 입력 비활성화
- 저장 버튼 숨김 또는 비활성화
- 안내 문구: "이 기능은 유료 플랜 전용입니다. 플랜 업그레이드 후 사용할 수 있습니다."

---

## 5. 설정 항목 상세

| 항목 | 필드명 | 타입 | 기본값 | 범위 |
|------|--------|------|--------|------|
| 기능 사용 여부 | `presenceCheckEnabled` | boolean | false | - |
| 오전 랜덤 확인 | `presenceCheckAmEnabled` | boolean | true | - |
| 오후 랜덤 확인 | `presenceCheckPmEnabled` | boolean | true | - |
| 확인 반경 | `presenceCheckRadiusMeters` | integer | 30 | 10~100 |
| 응답 제한 시간 | `presenceCheckResponseLimitMinutes` | integer | 20 | 5~60 |
| 실패 검토 처리 | `presenceCheckFailureNeedsReview` | boolean | true | - |

### 오전/오후 랜덤 구간
- 오전: 09:30 ~ 11:30
- 오후: 13:30 ~ 16:30

---

## 6. 저장 검증 규칙

- 유료 플랜인지 확인
- 반경 10~100 범위 확인
- 응답 제한 5~60 범위 확인
- 오전/오후 둘 다 OFF인데 enabled=true면: "오전 또는 오후 중 최소 1개는 활성화해야 합니다." 경고

---

## 7. 설정 변경 반영 시점

- 신규 스케줄 생성부터 반영
- 이미 생성된 당일 PENDING 요청은 유지
- OFF 후 신규 요청부터 중단

---

## 8. 관리자 안내 문구

> - 이 기능은 유료 플랜 전용입니다.
> - 오전/오후 랜덤 시각에 체류 확인 알림이 발생합니다.
> - 근로자가 확인 버튼을 누르면 현재 위치가 현장 반경 내인지 검사합니다.
> - 실패 또는 미응답 건은 검토 대상으로 분류될 수 있습니다.

---

## 9. 향후 확장 고려

- 현장별 override 설정
- 특정 현장 제외
- 특정 직군 적용
- 하루 1회/2회 선택

현재는 노출 안 하되, 구조는 확장 가능하게 설계.

---

## 10. 최종 명세 문구

> 관리자 설정 화면에서는 중간 체류 확인 기능의 사용 가능 여부, 활성화 여부, 오전/오후 랜덤 사용 여부, 반경, 응답 제한 시간, 실패 시 검토 처리 여부를 관리할 수 있어야 한다. 무료 플랜에서는 모든 설정을 비활성화하고, 유료 플랜에서만 실제 설정 변경을 허용한다.
