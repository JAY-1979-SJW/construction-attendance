# 중간 체류 확인 기능 — 상품/권한 정책 상세 지시문

> 기준일: 2026-03-21
> 대상 시스템: attendance.haehan-ai.kr
> 기능명: 중간 체류 확인(Presence Check)

---

## 1. 정책 목적

중간 체류 확인 기능은 기본 출퇴근 기능과 다르다.
허위 출퇴근 억제, 체류 개연성 보완, 운영 정밀도 향상 목적의 **추가 기능**이다.

- 유료 플랜 전용 기능
- 관리자 설정형 기능
- 기본값 OFF
- 조직 운영 정책에 따라 선택 적용

---

## 2. 기능 포지셔닝

| 구분 | 내용 |
|------|------|
| 기본 출퇴근 기능과의 관계 | 대체 아님, 보완 기능 |
| 기능 성격 | 노무 관리 강화 / 체류 개연성 검증 |
| 과금 성격 | 프리미엄 유료 기능 |

---

## 3. 플랜 정책

### 무료 플랜

- 기능 사용 불가
- 관리자 설정 화면 비활성 표시
- 스케줄 생성 안 함
- 응답 API 차단
- 근로자 앱 미노출

표시 문구: "이 기능은 유료 플랜 전용입니다."

### 유료 플랜

- 기능 사용 가능
- **관리자가 활성화해야만 실제 동작**
- 기본값 OFF

---

## 4. 활성화 정책

기능 동작 조건 (AND):
1. 조직이 유료 플랜일 것
2. 관리자가 기능을 활성화했을 것

---

## 5. 적용 범위 정책

### 1차 구현 범위

- 조직 전체 단위 ON/OFF
- 조직 단위 오전/오후 설정
- 조직 단위 반경/응답시간 설정

### 2차 확장 범위 (설계만 고려)

- 현장별 ON/OFF
- 현장별 반경 override
- 근로자 그룹별 ON/OFF
- 특정 직군/협력업체만 적용

---

## 6. 기본 설정값

유료 플랜 조직의 초기 기본값:

| 항목 | 기본값 |
|------|--------|
| 관리자 활성화 | false |
| 오전 랜덤 사용 | true |
| 오후 랜덤 사용 | true |
| 반경 | 30m |
| 응답 제한 시간 | 20분 |
| 실패/미응답 검토 처리 | true |

---

## 7. 검토 대상 처리 정책

중간 체류 확인은 출퇴근을 취소하는 기능이 아니다.

- 출퇴근 기록 유지
- 자동 결근 처리 안 함
- 실패/미응답 → `needsReview` 분류
- 노임 집계 전 검토 대상 포함
- 반복 시 관리자 주의 대상

---

## 8. 권장 도입 대상

**권장**
- 허위 출퇴근 우려가 높은 현장
- 다중 현장 이동이 잦은 조직
- 노무 증빙 정밀도가 중요한 유료 고객
- 원청/발주처 보고용 근태 통제가 필요한 경우

**비권장**
- 소규모 단순 현장
- 근로자 앱 적응 초기 단계
- 파일럿 초기 / QR 출퇴근 미안정 조직

---

## 9. 최종 정책 문구

> 중간 체류 확인 기능은 유료 가입 조직에 한해 제공하는 관리자 설정형 프리미엄 기능으로 정의한다. 조직이 유료 플랜이더라도 관리자가 기능을 활성화한 경우에만 실제 동작하며, 오전 및 오후 랜덤 시각에 현장 체류 확인 알림을 발생시켜 현재 위치가 설정 반경 내인지 검사한다. 실패 또는 미응답 건은 자동으로 검토 대상으로 분류한다.
